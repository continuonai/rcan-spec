# RCAN v1.5 Planning

**Target:** Q3 2026 (before EU AI Act August 2026 deadline)

## Priority features

### P0 — EU AI Act compliance (Art. 13 Transparency)
- [ ] RCAN transparency disclosure message type (new MessageType for announcing AI model identity to users)
- [ ] Transparency manifest endpoint spec: GET /api/transparency
- [ ] Operator transparency obligations in rcan-config.json

### P1 — Safety hardening
- [ ] Hardware watchdog MCU integration protocol
- [ ] SIL/PLe hardware capability attestation via registry
- [ ] Multi-robot safety coordination (fleet-level ESTOP propagation)

### P2 — Federation
- [ ] Authenticated federation between RRF nodes
- [ ] Robot-to-robot direct RCAN channels (peer-to-peer, no central broker)
- [ ] Fleet RCAN mesh networking spec

### P3 — Developer experience
- [ ] RCAN OpenAPI spec for all REST endpoints
- [ ] rcan-go SDK (Go language binding)
- [ ] RCAN CLI tool (rcan-cli) for testing and inspection
- [ ] Simulator protocol (rcan-sim) for virtual robot testing

## Open issues to resolve for v1.5
- #159 EU AI Act compliance (August 2026 deadline)
- Governance charter finalization

## Timeline
- April 2026: P0 transparency spec draft
- May 2026: P1 safety hardening spec draft
- June 2026: P2 federation spec
- July 2026: v1.5 RC + SDK updates
- August 2026: v1.5 release (before EU AI Act deadline)
